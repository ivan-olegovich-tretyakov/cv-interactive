## Part 12: What to avoid

### LinkedIn automation traps

- **Never automate connection requests, messaging, or Easy Apply.** LinkedIn's behavioral AI in 2026 detects non-human patterns (perfectly timed actions, linear activity graphs). Browser-based automation carries 60% higher detection risk. Account restrictions are tiered: 24-hour lockout â†’ 3-14 day lock + ID verification â†’ permanent ban with <15% recovery rate.
- **LinkedIn explicitly prohibits:** "bots or other unauthorized automated methods to access the Services, add or download contacts, send or redirect messages, create, comment on, like, share, or re-share posts, or otherwise drive inauthentic engagement."
- **The LinkedIn API is not a loophole.** Personal profile posting (r_member_social) is a CLOSED permission â€” LinkedIn is not accepting access requests. The Community Management API requires a registered legal entity. The free Share API is extremely limited.

### Mass-apply tools

Tools like Sonara, LoopCV, LazyApply, or the mass-apply features of any skill will damage your brand at executive level. Every application should be targeted and personalized.

### OpenClaw security

- Never install on your personal machine. VPS only.
- Vet every skill before installing (check VirusTotal report on ClawHub, read source code).

- Never let Ironclaw or any browser skill use your personal Chrome profile. Create a dedicated, minimal profile if browser features are needed.
- Keep OpenClaw behind a firewall, never exposed to the public internet.
